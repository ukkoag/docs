Analytics API

IMPORTANT​

This is a proposed API spec. The actual API spec might differ from this, although not heavily.

Overview​

Error Reporting and Debugging​

We strongly recommend setting X-REQUEST-ID header to UUID in each request. This will help us debug requests that failed. X-REQUEST-ID header will also be set in response.

Data Format​

Timestamp should be in ISO 8601 format and in UTC timezone.

Get Disease Risk Values​

GET https://external.ukkoag.com/v0.1/disease/<management_unit_uuid>/<model_name>?start_timestamp=iso_format&end_timestamp=iso_format

Request​

management_unit_uuid: 
Must include the JWT token obtained in the bearer authentication header of the request.

model_name:
NOT optional.

start_timestamp:

end_timestamp:


Returns disease risk data between `start_timestamp` and `end_timestamp`. Both ends are INCLUSIVE.
`end_timestamp` can be nullable. In that case the API will return all the data upto the current hour.
If both `start_timestamp` and `end_timestamp` are absent, then the readings for the most recent
24 hours will be returned.

Response​

`manamgent_unit_uuid`: ID of the management unit.

Hourly/daily (depending on the disease model) disease risk. If it's hourly the time will be in UTC timezone. If it's daily then it will be date in local timezone.

NOTE: Disease risk values are NOT guaranteed to be immutable or non-nullable. As the risk depends on the weather forecast.
